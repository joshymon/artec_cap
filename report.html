1. About

This section describes the Hana Cloud Compatible DB Artifacts migrated from XS Advanced, as well as any limitations or differences in behavior, where applicable.

2.Migration Status

HANA native artifacts such as 'hdbtable', 'hdbcalculationviews', 'hdbview', and 'hdbfunctions' will not be read by CDS. To make these objects recognizable to CDS, respective proxy files are created with certain limitations. For more information on this,please check the Hana Cloud Documentation.

Proxy entities for CalculationViews are created in their respective folders under the CDS folder. However, for hdbtables, hdbviews, and hdbfunctions, the proxies are created in the CDS folder with the prefix 'Proxy_' followed by the entity name and the extension '.cds'.

In calculation Views unsupported data types date(), day(), time() and counter datatype ( Integer ) are converted into respective supported data types daydate(), daytime(), secondtime() and counter datatype ( BIGINT ) .

While transitioning to CDS, the migration process involves renaming tables. Consequently, all other HANA artifacts that use these tables or views are also updated.

Table type definitions are not supported in Hana Cloud. Hence, during the migration, respective .hdbtabletype definitions are created by the extension and placed in the types folder in the target project.

In calculation views with parameters, the parameter's datatype must be defined as a variable property. If it is not, proxy calculation views will default to using the string datatype for this parameter. If the string datatype is not correct, it must be changed manually.

Series entity is not supported in Hana Cloud. These will be converted to regular entities.

If the keyword 'KEY' is being used as a key in cds entity definitions, the extension will change it to '![key]'.

The following features are currently out of scope:

  • Converting xsodata into cap service definition
  • Converting “xsjs","xsjslib" into cap nodejs
  • Creating proxy cds for cross container schema
  • Following Artifacts are not currently supported '.hdbreptask', '.hdbvirtualtable', '.hdbflowgraph'

4.Post Migration Steps

Migration of DB Artifacts from XS to CAP is complete. However, please verify these steps post migration:

  • SQL syntax changes in procedure, ex: UPDATE FROM has to be changed into MERGE INTO, TRUNCATE statement with a DELETE FROM statement

  • Flowgraph, Reptask and Replication artifacts Changes are not covered.

  • If Aliases are being used anywhere in the CDS files, please make sure that these are in uppercase.

  • A folder named 'unsupported_feature' has been created by the extension to contain file extensions that are not supported in HANA Cloud. The files in this folder need to be handled manually.

  • Unsupported types and functions in calculation view ex: "CE_FUNCTION", "CACHE" etc, please check the Hana Cloud documentation

  • Verify the cross container artifacts before deployment

  • Series entity is not supported in Hana Cloud so they will be removed by the extension. Please check Migration Documentation for more information.

  • CDS Access Policy entity definition is not supported in CAP CDS.

  • For HdbSynonym, HdbSynonymconfig and Hdbrole files, please check target object parameters before deployment.

  • The Memory threshold parameter is not supported in Hana Cloud and will be removed by the extension.

  • The extension creates indexes for column store tables. Indexes for row store tables must be manually created.

  • FuzzySearch and Index have to be manually created.
